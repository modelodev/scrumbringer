## Anti-Patterns Verdict

No parece una galería de UI generada por IA: hay arquitectura real, tokens, dark mode, skip link, componentes reutilizables y estados remotos. El problema técnico principal es deuda acumulada: side stripes como lenguaje de identidad/estado, sombras+bordes repetidos, uppercase labels en muchas superficies y patrones admin/table que no siempre tienen soporte de teclado/ARIA completo.

### [P1] Colores semánticos light no pasan AA como texto pequeño

Location:
- `apps/client/src/scrumbringer_client/theme.gleam:133`
- `apps/client/src/scrumbringer_client/theme.gleam:134`
- `apps/client/src/scrumbringer_client/theme.gleam:135`
- `apps/client/src/scrumbringer_client/styles/ux.gleam:292`

Category: Accessibility / Theming

Evidence:
- `--sb-warning` on `--sb-surface`: 3.19:1
- `--sb-success` on `--sb-surface`: 3.30:1
- `--sb-info` on `--sb-surface`: 4.10:1
- AA small text requires 4.5:1.

Impact: badges, status text and small state indicators can be hard to read in light theme, especially for low-vision users.

WCAG/Standard: WCAG 2.2 SC 1.4.3 Contrast (Minimum).

Recommendation: Split semantic tokens into background/border/text roles, using darker text tokens in light mode and brighter accents only for fills/borders.

Suggested command: `$impeccable colorize apps/client/src/scrumbringer_client`

## Patterns & Systemic Issues

- Token drift: naming split between `danger` and `error`, plus missing `hover` and `accent`.
- Accessibility is present but uneven: skip link, focus-visible and many aria labels exist, but reusable components do not enforce them everywhere.
- Tables are structurally reusable but not yet fully accessible as sortable/interactive data grids.
- Mobile has serious effort behind it, but authenticated mobile flows need runtime validation.
- State/identity color semantics are mixed through side stripes, badges, dots and tinted surfaces.

## Positive Findings

- `apps/client` compiles with `gleam check`.
- Impeccable detector returned no static AI-slop findings.
- Login snapshot exposes sensible accessible names: skip link, h1, email/password required inputs, named buttons.
- Global `:focus-visible` exists and reduced-motion fallback is present.
- Light/dark theme tokens exist and most body/muted text contrast is strong.
- Data table mobile collapse uses `data-label`, which is a good base for responsive tables.
- Shared components exist for action buttons, dialogs, form fields, error notices, badges, skeletons and tables.
