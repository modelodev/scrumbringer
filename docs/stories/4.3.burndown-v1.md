# Story 4.3: Burndown v1 (fichas en sprint)

## Status: Draft

## Story
**As a** project admin,
**I want** a burndown view for an active sprint,
**so that** I can understand progress over time.

## Acceptance Criteria
1. **Sprint concept exists**: A sprint exists with at least `{ id, project_id, name, end_date? }`.
2. **Scope defined**: Sprint contains fichas with a numeric `work_units`.
3. **Burndown computed**: Burndown series uses a configurable “done state” for fichas.
4. **UI**: The sprint board shows burndown with a simple chart or table.
